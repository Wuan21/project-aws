Frontend Hosting with S3 & CloudFront

Frontend Hosting with S3 & CloudFront (OAC)

In this step, you will set up static website hosting for the frontend application. We will use Amazon S3 to store the files and Amazon CloudFront to distribute them globally using Origin Access Control (OAC) to secure the bucket from direct public access.

1. Create S3 Bucket

  1. Open the Amazon S3 console.
  2. Select General purpose buckets on the left menu, then click Create bucket.
  3. Fill in the Create bucket form:
    • Bucket name: Enter webquiz-dev-frontend-YOUR_ACCOUNT_ID (replace YOUR_ACCOUNT_ID with your actual AWS Account ID).
    • Object Ownership: Select ACLs disabled / Bucket owner enforced (default).
    • Block Public Access settings for this bucket: Check ✅ Block all public access (default).
    • Bucket Versioning: Select Enable.
    • Click Create bucket.

2. Create CloudFront Distribution with OAC

  1. Open the Amazon CloudFront console.
  2. Click Create distribution.
  3. Step 1 - Origin:
    • Origin domain: Select the S3 bucket you just created (webquiz-dev-frontend-YOUR_ACCOUNT_ID.s3.amazonaws.com).
    • Origin access: Select Origin access control settings (recommended).
    • Click Create control setting:
      • Keep name defaults and ensure signing behavior is set to Sign requests (recommended).
      • Click Create.
  4. Step 2 - Default cache behavior:
    • Viewer protocol policy: Select Redirect HTTP to HTTPS.
    • Allowed HTTP methods: Select GET, HEAD, OPTIONS.
  5. Step 3 - Settings:
    • Default root object: Enter index.html.
  6. Click Create distribution.
  7. Once created, a warning banner will appear. Copy the generated S3 Bucket Policy text provided by CloudFront (you will paste this into S3 in the next step).

3. Configure S3 Bucket Policy

  1. Go back to your S3 bucket webquiz-dev-frontend-YOUR_ACCOUNT_ID in the S3 console.
  2. Select the Permissions tab.
  3. Scroll down to Bucket policy and click Edit.
  4. Paste the policy generated by CloudFront (which permits CloudFront to read objects from your bucket):
    {
    "Version": "2018-10-17",
    "Statement": [
        {
            "Sid": "AllowCloudFrontOAC",
            "Effect": "Allow",
            "Principal": {
                "Service": "cloudfront.amazonaws.com"
            },
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::webquiz-dev-frontend-YOUR_ACCOUNT_ID/*",
            "Condition": {
                "StringEquals": {
                    "AWS:SourceArn": "arn:aws:cloudfront::YOUR_ACCOUNT_ID:distribution/YOUR_DISTRIBUTION_ID"
                }
            }
        }
    ]
}
    (Ensure you replace YOUR_ACCOUNT_ID and YOUR_DISTRIBUTION_ID with your actual IDs).
  5. Click Save changes.

4. Configure Single Page Application (SPA) Routing

To ensure client-side routing works correctly without throwing 403 Access Denied errors on page reloads:

  1. Go back to your CloudFront distribution details.
  2. Select the Error pages tab.
  3. Click Create custom error response for two errors:
    • Error 1:
      • HTTP error code: Select 403: Forbidden.
      • Customize error response: Select Yes.
      • Response page path: Enter /index.html.
      • HTTP response code: Select 200: OK.
      • Click Create.
    • Error 2:
      • HTTP error code: Select 404: Not Found.
      • Customize error response: Select Yes.
      • Response page path: Enter /index.html.
      • HTTP response code: Select 200: OK.
      • Click Create.

5. Build and Deploy the Frontend

  1. Open your frontend source code folder on your local machine.
  2. Configure your environment file (e.g., .env or config.js) with the AWS values you saved:
    const config = {
    COGNITO_USER_POOL_ID: "ap-southeast-1_xxxxxxxxx",
    COGNITO_APP_CLIENT_ID: "xxxxxxxxxxxxxxxxxxxxxxxxxx",
    REST_API_ENDPOINT: "https://xxxxxxxxxx.execute-api.ap-southeast-1.amazonaws.com/dev",
    WEBSOCKET_API_ENDPOINT: "wss://xxxxxxxxxx.execute-api.ap-southeast-1.amazonaws.com/dev"
};
  3. Build the application static assets:
    npm run build
  4. Upload the built static files (found inside the dist/ or build/ directory) directly into your S3 bucket using the AWS S3 Console or the AWS CLI:
    aws s3 sync dist/ s3://webquiz-dev-frontend-YOUR_ACCOUNT_ID
  5. In the CloudFront distribution overview page, copy the Distribution Domain Name (e.g., https://xxxxxxxxxx.cloudfront.net) and access it in your browser to test!
